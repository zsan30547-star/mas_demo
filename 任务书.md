## 1. 项目概述

### 1.1 项目名称

**多模型智能体协作平台**（Multi-Model Agent Collaboration Platform）

### 1.2 一句话

一个支持多模型混合编排的智能体协作平台——通过表单配置 Agent、自定义执行步骤，Langgraph 负责编排调度，让不同 AI 模型各司其职、协作完成任务。

### 1.3 项目背景

当前大模型领域百花齐放，不同模型各有所长：

| 模型 | 优势 | 适合角色 |
|------|------|----------|
| **DeepSeek** | 推理规划能力强 | 任务规划 |
| **Qwen-VL** | 多模态能力 | 图片 OCR / 视觉理解 |
| **Gemini** | 执行能力强 | 内容生成 / 任务执行 |
| **Claude** | 安全审慎 | 质量验证 / 纠错 |

单一模型难以覆盖所有场景。现有平台（如 Dify、Langflow）虽支持多模型，但偏向拖拽式工作流搭建，学习成本高。本项目旨在提供一个**轻量级的多模型智能体协作平台**，通过表单配置 Agent、步骤编排工作流，让用户能以最低成本利用多个模型的互补优势。

### 1.4 项目目标

1. **多 Agent 管理**：通过表单配置不同模型驱动的 Agent（视觉/规划/执行/验证/搜索）
2. **灵活编排**：用户自定义 Agent 执行顺序，支持步骤拖拽排序
3. **智能协作**：Langgraph Supervisor Pattern 编排多 Agent 协作流程
4. **长期记忆**：Chroma 存储 Agent 对话历史和知识库，支持 RAG
5. **实用演示**：以「智能研究报告生成」为演示场景，展示完整能力


```
pending → running → success

```

## 5. Agent 设计

### 5.1 预置 Agent 模板

系统启动时自动创建以下 5 个预置 Agent：

| Agent | 默认模型 | 角色提示词摘要 |
|-------|---------|--------------|
| **规划专家** | DeepSeek | 你是一个任务规划专家，负责将复杂任务拆解为可执行的子步骤并规划执行顺序 |
| **视觉识别专家** | Qwen-VL | 你是一个视觉分析专家，擅长从图片中提取文字信息、识别物体和场景 |
| **内容执行专家** | Gemini | 你是一个内容执行专家，根据任务规划和参考资料执行具体任务，如撰写、分析、计算 |
| **质量验证专家** | Claude | 你是一个质量验证专家，负责检查结果的准确性、完整性和逻辑一致性 |
| **信息搜索专家** | DeepSeek + SerpAPI | 你是一个搜索专家，根据需求搜索最新信息并整理汇总 |

### 5.2 Agent 抽象接口

每个 Agent 继承自统一的 BaseAgent：

```python
class BaseAgent(ABC):
    """Agent 基类，所有 Agent 必须实现 run 方法"""
    
    def __init__(self, config: AgentConfig):
        self.config = config
        self.gateway = ModelGateway(config)
    
    @abstractmethod
    async def run(self, input_text: str, context: dict = None) -> AgentResult:
        """执行 Agent 任务"""
        pass
```

### 5.3 Agent 处理器详情

#### 规划 Agent (Planner)

```
输入: 用户课题描述（如 "2026年AI Agent发展趋势"）
处理: DeepSeek 分析并拆解为子任务列表
输出: JSON 格式的子任务列表
      [{ "task": "搜索市场规模数据", "priority": 1 }, ...]
```

#### 视觉 Agent (Vision)

```
输入: 图片 URL / Base64 + 指令
处理: Qwen-VL 多模态识别
输出: 提取的文本信息 / 图片描述
```

#### 执行 Agent (Executor)

```
输入: 上一步输出的资料 + 执行指令
处理: Gemini 按指令执行
输出: 完成的内容（报告/分析/代码等）
```

#### 验证 Agent (Validator)

```
输入: 待验证的内容 + 质量标准
处理: Claude 逐项检查
输出: 修正后的版本 + 验证报告（问题列表 + 修改建议）
```

#### 搜索 Agent (Search)

```
输入: 搜索关键词 / 问题
处理: SerpAPI 搜索 → LLM 整理摘要
输出: 结构化搜索结果（标题+摘要+来源）
```

## 6. 工作流编排

### 6.1 编排模型

用户通过**步骤编辑器**创建工作流，每个步骤是一个 Agent 调用，步骤间顺序执行，上一步的输出作为下一步输入的上下文。

### 6.2 步骤数据流

```
                 输入模板引用
用户输入 ──────────────────────────┐
    │                              │
    ▼                              ▼
┌─────────┐   ┌─────────┐   ┌─────────┐
│ Step 1  │→→→│ Step 2  │→→→│ Step 3  │
│ Planner │   │ Search  │   │Executor │
└────┬────┘   └────┬────┘   └────┬────┘
     │             │             │
     ▼             ▼             ▼
  step1.output  step2.output  step3.output
```

### 6.3 步骤输入模板语法

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{input}}` | 用户提交的原始输入 | "分析以下课题：{{input}}" |
| `{{step1.output}}` | 第1步 Agent 的输出 | "基于以下资料：{{step1.output}}" |
| `{{stepN.output}}` | 第N步 Agent 的输出 | — |

### 6.4 预置工作流模板

**研究报告生成**（演示场景）：

```
名称: 研究报告生成
描述: 输入研究课题，自动规划→搜索→撰写→验证，输出完整报告
步骤:
  1. 规划专家: "将以下课题拆解为研究子任务：{{input}}"
  2. 搜索专家: "搜索以下主题的最新信息：{{step1.output}}"
  3. 执行专家: "基于以下资料撰写完整研究报告：{{step2.output}}"
  4. 验证专家: "检查以下报告的质量并修正：{{step3.output}}"
```

## 7. 数据库设计

### 7.1 整体 ER 概览

```
sys_user (1) ──── (N) agent_config
sys_user (1) ──── (N) workflow_template
sys_user (1) ──── (N) task

workflow_template (1) ──── (N) task

task (1) ──── (N) task_step_log
task_step_log (N) ──── (1) agent_config (可为空，保留历史快照)
```

### 7.2 建表 SQL

```sql
-- ===================================================
-- 多模型智能体协作平台 - 数据库初始化脚本
-- ===================================================

CREATE DATABASE IF NOT EXISTS agent_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE agent_platform;

-- 用户表
CREATE TABLE sys_user (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password    VARCHAR(255) NOT NULL COMMENT '密码(BCrypt)',
    email       VARCHAR(100)          COMMENT '邮箱',
    avatar      VARCHAR(500)          COMMENT '头像URL',
    status      TINYINT      DEFAULT 1 COMMENT '1-启用 0-禁用',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- Agent 配置表
CREATE TABLE agent_config (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id         BIGINT       NOT NULL COMMENT '所属用户',
    name            VARCHAR(100) NOT NULL COMMENT 'Agent名称',
    agent_type      VARCHAR(50)  NOT NULL COMMENT '角色类型: vision/planner/executor/validator/search',
    model           VARCHAR(100) NOT NULL COMMENT '模型名称',
    system_prompt   TEXT                  COMMENT '角色提示词(System Prompt)',
    temperature     DECIMAL(2,1) DEFAULT 0.7 COMMENT '生成温度',
    max_tokens      INT          DEFAULT 4096 COMMENT '最大Token数',
    endpoint        VARCHAR(500)          COMMENT 'API端点地址',
    api_key_enc     VARCHAR(500)          COMMENT '加密后的API Key (AES-256-CBC)',
    icon            VARCHAR(50)           COMMENT '图标标识',
    sort_order      INT          DEFAULT 0 COMMENT '排序号',
```

## 9. Langgraph 执行引擎

### 9.1 核心架构

```
用户编排的步骤 JSON:

steps = [
  { "agentId": 1, "order": 1, "inputTemplate": "拆解任务：{{input}}" },
  { "agentId": 5, "order": 2, "inputTemplate": "搜索：{{step1.output}}" },
  { "agentId": 3, "order": 3, "inputTemplate": "撰写：{{step2.output}}" }
]

         ↓ 动态构建 Langgraph StateGraph

┌───────────────────────────────────────────────────┐
│  StateGraph(TaskState)                             │
│                                                    │
│  START → step1_node → step2_node → step3_node → END│
│         ↑           ↑           ↑                  │
│      Planner     Search     Executor               │
│      (DeepSeek)  (SerpAPI)  (Gemini)               │
└───────────────────────────────────────────────────┘

         ↓ compile & invoke(input)

TaskState 在各个节点间传递和更新
```

### 9.2 TaskState 定义

```python
from typing import TypedDict, List, Dict, Any, Optional

class TaskState(TypedDict):
    """Langgraph 工作流状态"""
    task_id: str                          # 任务ID
    user_input: str                       # 用户原始输入
    files: List[Dict]                     # 上传文件
    steps_config: List[Dict]              # 步骤配置
    current_step: int                     # 当前步骤索引
    step_outputs: Dict[int, str]          # { step_index: output_text }
    final_output: str                     # 最终输出
    step_logs: List[Dict]                 # 步骤执行日志
    errors: List[str]                     # 错误信息
    memory_context: Optional[str]         # Chroma 检索的历史记忆
    knowledge_context: Optional[str]      # Chroma 检索的知识库内容
```

### 9.3 节点处理器

```python
# 通用步骤节点 - 根据步骤配置动态调用对应 Agent
async def step_node(state: TaskState) -> dict:
    """执行单个步骤"""
    step_idx = state["current_step"]
    step_config = state["steps_config"][step_idx]
    agent = AgentFactory.create(step_config["agentId"])
    
    # 渲染输入模板
    input_text = render_template(
        step_config["inputTemplate"],
        state["user_input"],
        state["step_outputs"]
    )
    
    # 可选: 从 Chroma 检索上下文
    context = await chroma_service.retrieve_context(input_text)
    
    # 调用 Agent
    result = await agent.run(input_text, context=context)
    
    # 记录步骤日志
    step_log = {
        "step_index": step_idx + 1,
        "agent_name": step_config.get("agentName", ""),
        "input": input_text,
        "output": result.output,
        "status": "success" if result.success else "failed",
        "duration_ms": result.duration_ms
    }
    
    return {
        "current_step": step_idx + 1,
        "step_outputs": {**state["step_outputs"], step_idx + 1: result.output},
        "step_logs": [*state.get("step_logs", []), step_log],
        "errors": [*state.get("errors", []), *([] if result.success else [result.error])]
    }
```

### 9.4 条件路由

```python
# 路由函数 - 决定下一步执行还是结束
def route_next(state: TaskState) -> str:
    """根据当前步骤索引决定路由"""
    if state["current_step"] >= len(state["steps_config"]):
        return END
    if state["errors"]:
        return END  # 出错了就终止
    return "step_node"
```

### 9.5 动态构图

```python
from langgraph.graph import StateGraph, START, END

def build_workflow(steps_config: List[Dict]) -> CompiledStateGraph:
    """根据用户编排的步骤动态构建 Langgraph 工作流"""
    
    workflow = StateGraph(TaskState)
    
    # 添加通用步骤节点（所有步骤共用同一个节点处理器）
    workflow.add_node("step_node", step_node)
    
    # 起始边
    workflow.add_edge(START, "step_node")
    
    # 条件边：每次执行完后判断是否还有下一步
    workflow.add_conditional_edges(
        "step_node",
        route_next,
        {
            "step_node": "step_node",  # 继续下一步
            END: END                    # 执行完毕
        }
    )
    
    return workflow.compile()
```

### 9.6 整体执行流程

```
1. Spring Boot 收到任务提交请求
2. 存入 MySQL (task.status = "pending")
3. 发送 MQ 消息 (包含 taskId + workflowSteps + input)
4. FastAPI 消费 MQ 消息
5. 更新 Redis task:status → "running"
6. 动态构建 Langgraph StateGraph (build_workflow)
7. 更新 Redis task:status → "running"
8. 调用 app.invoke() 开始执行
9. 每步执行:
   a. 通过 Model Gateway 调用 LLM
   b. 可选检索 Chroma 记忆/知识
   c. 写入 task_step_log
   d. 更新 Redis task:steps 进度
10. 执行完毕:
    a. 回写 task.final_output
    b. 更新 task.status → "completed"
    c. 存入 Chroma 记忆
    d. 推送 Redis Pub/Sub 通知
11. Spring Boot 任务状态轮询/订阅获取最新状态
12. 前端展示结果
```

## 14. 演示场景

### 14.1 智能研究报告生成

**场景描述**：

用户输入一个研究课题，系统通过多 Agent 协作自动生成一份结构化的研究报告。

**步骤演示**：

```
用户输入: "2026年AI Agent在企业服务中的应用趋势"
上传附件: 可选 PDF 参考文档
点击: "执行"
```

**Agent 协作过程**：

| 步骤 | Agent | 模型 | 执行内容 | 预期输出 |
|------|-------|------|----------|----------|
| 1 | 规划专家 | DeepSeek | 分析课题，拆解为子任务 | 4-5个研究子任务列表 |
| 2 | 搜索专家 | DeepSeek + SerpAPI | 搜索每个子任务的最新信息 | 搜索结果摘要 |
| 3 | 执行专家 | Gemini | 基于资料撰写完整报告 | 3000字+研究报告 |
| 4 | 验证专家 | Claude | 检查事实、结构、语法 | 修正后的最终版报告 |

**前端展示**：

1. 实时进度条（步骤 2/4）
2. 每个步骤可展开查看输入/输出
3. 最终报告 Markdown 渲染
4. 步骤耗时统计

### 14.2 扩展场景（用户可自定义）

用户可创建自己的工作流模板，例如：

- **智能票据识别**：视觉(Qwen-VL) → 规划(DeepSeek) → 执行(Gemini) → 验证(Claude)
- **竞品分析报告**：搜索 → 规划 → 执行 → 验证
- **多语言翻译审校**：执行(Gemini翻译) → 验证(Claude审校)

## 附录

### A. 快速启动命令

```bash
# 1. 启动基础设施
docker-compose up -d

# 2. 启动 Spring Boot 后端
cd backend
mvn spring-boot:run

# 3. 启动 FastAPI AI 引擎
cd ai-engine
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# 4. 启动 Vue3 前端
cd frontend
npm install
npm run dev
```

### B. Docker Compose 配置

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: agent_platform
    ports:
      - "3306:3306"
    volumes:
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
      - mysql_data:/var/lib/mysql

  redis:
    image: redis:7
    ports:
      - "6379:6379"

  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"
    environment:
      RABBITMQ_DEFAULT_USER: admin
      RABBITMQ_DEFAULT_PASS: admin123

  chroma:
    image: chromadb/chroma:latest
    ports:
      - "8001:8000"
    volumes:
      - chroma_data:/chroma/chroma

volumes:
  mysql_data:
  chroma_data:
```
